# S.T STAR — Accounting & Operations System (Backend)

Back-office system for **S.T STAR Logistics & Customs Clearance Service Co., Ltd** covering
accounting, invoicing, customs-clearance operations, and administration. This document
summarizes what is built and working today.

---

## What the system does today

### 🧾 Invoicing & Receivables
- **Tax invoices and debit notes** with automatic sequential numbering (e.g. `ST25-000028`).
- Multiple line items per invoice, with **10% VAT** applied automatically (rate configurable;
  debit notes are VAT-exempt).
- Full invoice lifecycle: **Draft → Issued → Partially Paid → Paid** (plus Overdue,
  Cancelled, Refunded).
- **Payment recording** against invoices in 6 methods: cash, bank transfer, check,
  credit card, mobile money, cryptocurrency — with automatic balance-due tracking.
- **Accounts receivable aging report** showing which invoices are overdue and by how long.

### 📚 Accounting (double-entry bookkeeping)
- **Chart of accounts** with 17 pre-configured GL accounts: revenue (4001–4005),
  expenses (5001–5012), bank, receivables, payables, VAT payable, deposits.
- **Journal entries** with enforced balance — every entry's debits must equal its credits.
- **Automatic posting**: approving an income record, expense, or invoice automatically
  creates the correct journal entry (e.g. debit Bank / credit Revenue).
- Manual journal entries and adjustments for the accountant.
- Account balances updated in real time.

### 💰 Income & Expense records
- Auto-numbered records (`INC-2026-0001`, `EXP-2026-0001`).
- Income classified by **service type**: customs clearance, storage fee, license fee,
  trucking, logistics, bonded warehouse, brokerage, and more.
- Expenses classified by type: technical operations, deposits paid, rental, utilities,
  office supplies, NSSF, withholding tax, bank charges, etc.
- **Approval workflow**: staff record the transaction → a manager or admin approves
  (or rejects with a reason) → the record is posted to the ledger and locked from editing.

### 🚢 Customs clearance operations
- **Clearance job register**: job number, date, customer, shipment status, BL/booking
  number, commodity, size/volume, port (PPAP / SHV / TL), import or export.
- Jobs can be linked to the income record generated by the shipment.

### 👥 Customers & Suppliers
- Master data with auto-generated customer/supplier codes (e.g. `027-01-26`).
- **Multilingual names** — English, Khmer, and Chinese.
- Tax ID (VAT registration), address, contacts, bank account details.
- Per-customer **payment terms** (days, discount).

### 🔒 Deposits
- Security deposits and advances from customers or to suppliers.
- Lifecycle tracking: **Active → Released / Returned / Forfeited**, tied to a GL account.

### 🏛️ Tax management
- Configurable **VAT rates** (10% default, 20% available).
- **Tax filing records** for VAT, income tax, withholding tax, NSSF, quarterly and
  annual filings — with Draft → Filed → Accepted status tracking.

### 📊 Financial reports
- **Profit & Loss**, **Balance Sheet**, income summary, expense summary,
  A/R aging, and tax summary.
- Monthly, yearly, or all-time views.
- **Export to PDF or Excel** for sharing and filing.

### 📎 Documents & attachments
- File attachments (invoices, receipts, supporting documents) on income, expense,
  and invoice records — stored securely in cloud storage (S3).

### 🕵️ Audit trail
- Every create, update, approve, reject, and delete is logged with the user, IP address,
  and timestamp — an immutable compliance log viewable by admins, accountants, and managers.

## Users & roles

The system has **5 user roles** with role-based access control on every endpoint:

| Role | What they can do |
|------|------------------|
| **ADMIN** | Everything — user management, company settings, tax rates, approvals, all financial operations |
| **ACCOUNTANT** | Invoices, income & expense records, payments, journal entries, chart of accounts, deposits, tax filings |
| **MANAGER** | Customers, suppliers, clearance jobs, **expense approval/rejection**, audit logs |
| **STAFF** | Record income and expenses (cannot approve them), day-to-day data entry |
| **VIEWER** | Read-only access |

Security: JWT token login, bcrypt-hashed passwords, token refresh, and per-endpoint
role guards. User accounts can be activated/deactivated by an admin.

---

## Key workflows

1. **Invoicing:** Accountant creates invoice (auto-numbered, VAT calculated) → finalizes
   it (posted to the ledger) → records customer payments → invoice status and balance due
   update automatically.
2. **Expenses:** Staff records an expense → manager or admin approves or rejects →
   on approval it is posted to the general ledger and locked.
3. **Income:** Staff records service revenue (clearance fee, storage, trucking…) →
   accountant/admin approves → automatic journal entry (debit Bank / credit Revenue).

## Company setup

The database is seeded with:
- Company profile: **S.T STAR Logistics & Customs Clearance Service Co., Ltd**,
  VAT ID `K009-STAR-0001`, Phnom Penh, Cambodia — editable by an admin in settings
  (name in EN/KH/CN, fiscal year start, default VAT rate, default payment terms, logo).
- The full chart of accounts (17 GL accounts).
- Two VAT rates (10% default, 20%).
- A default admin account (credentials configurable via the `SEED_ADMIN_EMAIL` /
  `SEED_ADMIN_PASSWORD` environment variables — change on first deployment).

## Technical overview

| | |
|---|---|
| Framework | NestJS 11 (TypeScript) |
| Database | PostgreSQL (Neon) via Prisma 7 |
| Auth | JWT (Passport), role-based guards |
| API docs | Swagger UI at `/docs` |
| Base URL | `http://localhost:3333/api` |
| Files | AWS S3 presigned uploads |
| Hardening | Helmet, CORS, global validation, health check at `/api/health` |

**20 feature modules:** auth, users, customers, suppliers, invoices, payments, income,
expense, journal, accounts, deposits, operations (clearance jobs), tax, reports,
settings, uploads, storage, audit, health, common.

### Running locally

```bash
pnpm install
pnpm start:dev      # dev server on :3333
pnpm build && pnpm start:prod
pnpm lint
pnpm test           # unit tests
pnpm test:e2e
```

Environment is validated at boot (`src/config/env.validation.ts`); the database URL is
configured in `prisma.config.ts`. Seed the database with `prisma/seed.ts`.
